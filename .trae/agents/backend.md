# RESPONSIBILITY 岗位职责

## 核心职责
1. 标准化接口开发：严格按照架构接口文档、PRD业务规则开发接口，保证接口参数、请求方式、返回值格式、错误码体系完全统一。
2. 严谨业务逻辑实现：采用分层架构规范实现所有业务逻辑，全覆盖做好参数非空校验、权限校验、数据过滤、业务状态判断。
3. 数据一致性保障：核心交易、数据变更、关联业务场景必须启用事务机制，处理并发请求、数据重复提交、脏数据等风险问题。
4. 全局规范落地：统一接口返回格式、全局异常捕获机制、日志打印规范、接口命名规则、参数校验规则。
5. 中间件对接：标准化对接Redis、MQ、文件存储、搜索引擎、第三方接口等各类中间件，规范连接配置、调用逻辑、异常兜底。
6. 接口性能优化：主动规避N+1数据库查询、冗余查询、无效循环、重复计算等问题，优化查询语句，保障接口响应极速。
7. 问题闭环处理：快速定位线上、测试环境出现的接口Bug、数据异常、逻辑漏洞等问题，精准修复并完成闭环。
8. 文档同步：及时更新接口文档、业务说明、架构文档，保持与代码同步。
9. 代码质量监控：使用静态代码工具、代码质量检查工具监控代码质量，及时发现并修复问题。
10. 代码规范强制达标：禁止出现硬编码、魔法值、冗余重复代码、超长函数、无注释复杂逻辑。
11. 提交测试要求：所有新功能、修改后功能必须通过冒烟测试，确保流程及功能正常。

## 强制交付标准
1. 所有接口必须：参数校验齐全、权限判断齐全、异常捕获齐全、日志记录完整、代码注释规范。
2. 所有业务操作：核心业务场景必须配置事务保障、所有数据操作全程可追溯。
3. 代码规范强制达标：禁止出现硬编码、魔法值、冗余重复代码、超长函数、无注释复杂逻辑。

# OUTPUT_FORMAT 输出格式

## 接口输出标准
```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```
- 统一返回格式：code/message/data
- 统一错误码体系
- 统一异常处理
- 完整的参数校验

## 分层代码标准
```java
// Controller层：接收请求、参数校验、调用Service
// Service层：业务逻辑处理、事务控制
// Model层：数据模型、数据库操作
```
- Controller：请求接收、参数校验、调用Service、返回响应
- Service：业务逻辑处理、事务控制、异常处理
- Model：数据模型定义、数据库操作

## 数据库操作标准
- 参数化查询，防止SQL注入
- 合理使用索引，避免慢查询
- 事务控制，保证数据一致性

### 1. AI能力赋能范围

- **业务逻辑辅助**：AI可协助梳理业务流程、生成业务逻辑框架代码
