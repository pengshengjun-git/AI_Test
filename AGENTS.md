任何AI角色开始任何工作前，必须按以下顺序完整阅读文档，不可跳过：


### 6.1 自动检测与创建机制（增强）

如果 memory 目录或文件不存在：

**AI 必须自动执行以下流程：**

1. **检测阶段**：检查 `/memory/` 目录是否存在，检查 memory/README.md 是否存在，检查核心 memory 文件是否完整
2. **创建阶段**：按顺序自动创建缺失的 memory 文件，每个文件包含标准模板结构
3. **填充阶段**：根据项目当前状态填充各 memory 文件

### 6.2 自动提醒机制（增强）

当 AI 检测到以下情况时，必须主动提醒：

1. **首次进入项目**：提示检查 memory 文件完整性
2. **架构变更时**：提醒更新 architecture.md
3. **接口变更时**：提醒更新 api-contract.md
4. **代码规范变更时**：提醒更新 coding-style.md
5. **发现问题时**：提醒更新 known-issues.md

# 十三、越权强制拦截与自动闭环（最高优先级，不可绕过）

## 1. 越权判定标准（满足任意一条=越权）
- 修改不属于本角色职责的文件
- 修改 **/memory/、**/agents/、AGENTS.md、trae_config.yaml
- 调用未在 trae_config.yaml allowed_tools 中声明的工具
- 跨角色决策、跨角色实现、跨角色修改
- 未获指派任务主动修改代码/架构/配置

## 2. 越权触发动作（AI必须严格执行，不可跳过）
一旦判定越权，AI 必须**立即、自动、无例外**执行以下步骤：
1. 立刻停止所有操作，输出：
   【违规告警】检测到越权操作，已自动停止并回滚
2. 自动回滚本次所有文件修改（恢复原状）
3. 生成《越权报告》，包含：
   - 越权角色
   - 尝试修改文件
   - 尝试调用工具
   - 违反规则条目
4. 强制移交对应责任人，格式：
   【当前角色：XXX，越权违规，已回滚，下一岗位：正确角色，开始工作】
5. 禁止再次尝试修改，必须等待正确角色处理

## 3. 并行排查规则
- 多个角色可**并行阅读、分析、定位问题**
- 但**只有职责归属角色能修改**，其他人只读
- 发现问题 → 报告 → 移交 → 等待处理
- 严禁：边排查边改、跨角色修复、擅自整改
