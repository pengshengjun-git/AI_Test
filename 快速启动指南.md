# 🚀 快速启动指南

## 为什么AI无法帮你启动服务？

因为当前AI环境的配置文件 `trae_config.yaml` 中，**所有Bash/Docker命令被禁止**，所以AI无法执行 `docker compose` 等命令。

## 你可以手动启动！

### 方式一：使用Windows批处理脚本（推荐）

双击运行：
```
AI角色系统一键部署.bat
```

或者进入services目录运行：
```
services\start-services.bat
```

### 方式二：使用Docker Compose命令（最简单）

打开PowerShell或CMD，进入项目目录，然后：

#### 1️⃣ 启动Mock演示模式（无需数据库，最快）
```bash
cd services
docker compose -f docker-compose.mock-demo.yml up -d --build
```

访问地址：**http://localhost:8080**

登录信息：
- 账号：`admin`
- 密码：`123456`

#### 2️⃣ 查看服务状态
```bash
docker compose -f docker-compose.mock-demo.yml ps
```

#### 3️⃣ 查看服务日志
```bash
docker compose -f docker-compose.mock-demo.yml logs -f
```

#### 4️⃣ 停止服务
```bash
docker compose -f docker-compose.mock-demo.yml down
```

### 方式三：完整开发模式（需要数据库）

如果你想启动完整版本（包括后端和数据库）：
```bash
cd deploy
docker compose -f docker-compose.dev.yml up -d
```

## 📋 服务端口说明

| 服务 | 端口 | 访问地址 |
|-----|------|---------|
| 前端 | 8080 | http://localhost:8080 |
| Mock服务 | 8020 | http://localhost:8020 |
| 后端（完整模式）| 8081 | http://localhost:8081 |

## 🔧 常见问题

### 1. Docker没运行？
- 先启动Docker Desktop
- 确保Docker Desktop正常运行

### 2. 端口被占用？
修改 `docker-compose.mock-demo.yml` 中的端口号

### 3. 构建失败？
尝试先拉取或重新构建：
```bash
cd services
docker compose -f docker-compose.mock-demo.yml build
```
