# AI角色系统使用指南

## 概述

这是一个企业级AI全流程协作系统，通过定义23个专业角色，实现AI自动化协作开发。本指南将帮助您：

- 在任何新项目中快速启用角色系统
- 让AI角色自动协作完成开发任务
- 遵循标准化的开发流程和规范

## 快速开始

### 方式一：复制文件到新项目（推荐）

1. **复制核心文件**
   ```bash
   # 复制到新项目根目录
   cp AGENTS.md /path/to/new-project/
   cp USE_GUIDE.md /path/to/new-project/
   
   # 复制角色定义目录
   cp -r agents /path/to/new-project/
   ```

2. **自动初始化memory系统**
   - AI会自动检测memory目录
   - 创建7个核心memory文件
   - 填充项目基本信息

3. **开始使用**
   ```
   【用户】启动企业级项目全流程开发：我要开发一个电商系统
   ```

### 方式二：从零开始创建

1. **创建AGENTS.md宪法文件**
   - 从本项目复制AGENTS.md
   - 放置到新项目根目录

2. **创建agents目录**
   ```bash
   mkdir -p agents
   ```

3. **创建角色定义文件**
   - 从本项目复制所有agents/*.md文件
   - 或使用默认模板创建

4. **AI自动创建memory**
   - AI检测到memory缺失
   - 自动创建完整的memory系统

### 2. Memory系统

Memory是项目的长期记忆，包含7个核心文件：

| 文件 | 作用 | 维护人 |
|------|------|--------|
| README.md | 系统总览 | 文档工程师 |
| architecture.md | 架构规范 | 系统架构师 |
| api-contract.md | API契约 | 后端开发 |
| coding-style.md | 代码规范 | 技术总监 |
| decisions.md | 技术决策 | 系统架构师 |
| changelog.md | 变更记录 | 文档工程师 |
| known-issues.md | 已知问题 | 测试工程师 |

**核心原则**：Memory First
- 开发前必须读取memory
- 决策后必须更新memory
- 禁止忽略历史架构

### 3. 标准化工作流

```
需求定义 → 架构设计 → UI设计 → 开发实现 → 安全审计 → 性能优化 → 测试验收 → 文档归档 → 部署上线
```

```
【当前角色：前端开发】完成注册页面开发

```

❌ **错误做法**：
```
【当前角色：前端开发】顺便把后端API也写了
```

✅ **正确做法**：

❌ **错误做法**：


## 常见问题

### Q1: 如何让AI自动创建memory？
**A**: AI会在首次进入项目时自动检测并创建。如果需要手动触发，可以说"初始化memory系统"

### Q2: 角色可以跳过吗？
**A**: 不能。工作流必须按顺序执行，除非明确不需要某个角色。

### Q3: 多个AI可以同时工作吗？
**A**: 可以，但每个AI必须扮演一个角色，不能跨角色。

### Q4: 如何自定义新角色？
**A**: 在agents/目录创建新文件，定义职责和边界，然后在AGENTS.md中添加。

### Q5: memory文件谁来维护？
**A**: 每个文件都有指定的维护责任人，负责及时更新。

### Q6: 违反规则会怎样？
**A**: AI会自动检测越权行为，停止操作并报告。
