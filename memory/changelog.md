## 记录规范
每条变更记录必须包含：
- 变更日期

## 变更类型定义
- **feat**：新功能
- **fix**：Bug修复
- **refactor**：重构
- **docs**：文档更新
- **style**：代码格式
- **perf**：性能优化
- **test**：测试相关
- **chore**：构建/工具

## 维护指南

### 添加新变更
1. 在顶部添加新条目
2. 包含完整的变更信息
3. 更新版本号
4. 标注责任人和审核人

### 版本号规范
- 主版本号：重大架构变更
- 次版本号：新功能新增
- 修订号：Bug修复和优化

格式：`v主版本.次版本.修订号`
示例：`v1.2.3`

### 风险等级定义
| 等级 | 说明 | 影响 |
|------|------|------|
| 低 | 轻微调整，不影响功能 | 无 |
| 中 | 部分功能受影响 | 局部 |
| 高 | 核心功能受影响 | 广泛 |

### 变更通知
- 高风险变更需要邮件通知
- 中风险变更需要在群聊通知
- 低风险变更仅记录文档
