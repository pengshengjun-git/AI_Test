## 三、通配符查找模式说明

### Glob通配符语法
| 模式 | 说明 | 示例 |
|------|------|------|
| `**` | 匹配任意层级目录 | `**/docs/*.md` 匹配任意目录下docs文件夹中的md文件 |
| `*` | 匹配任意字符（不含路径分隔符） | `test*.md` 匹配test开头的md文件 |
| `?` | 匹配单个字符 | `test?.md` 匹配test1.md, test2.md |
| `[]` | 匹配指定范围内字符 | `test[123].md` 匹配test1.md, test2.md, test3.md |

### 查找时的通配符组合使用示例
```
查找测试用例：
1. 首先尝试：**/docs/testing/测试用例文档.md
2. 然后尝试：**/docs/testing/测试用例*.md
3. 再尝试：**/测试用例*.md
4. 最后尝试：**/*test*.md
```

## 四、自动创建策略

如果通过所有通配符模式都找不到文档，按以下优先级创建：

### 目录自动创建规则
在创建文档前，必须先检查目标目录是否存在，如果不存在则自动创建：
- 检查并创建：`docs/`
- 检查并创建：`docs/testing/`
- 检查并创建：`docs/requirements/`
- 检查并创建：`docs/architecture/`
- 检查并创建：`docs/development/`
- 检查并创建：`docs/api/`
- 检查并创建：`docs/deployment/`
- 检查并创建：`docs/design/`

### 各类型文档创建位置优先级

#### 测试文档创建位置顺序
1. `docs/testing/` (优先标准位置)
2. `docs/` (次选位置)
3. 项目根目录 (最后选择)

#### 需求文档创建位置顺序
1. `docs/requirements/` (优先标准位置)
2. `docs/` (次选位置)
3. 项目根目录 (最后选择)

#### 开发文档创建位置顺序
1. `docs/development/` (优先标准位置)
2. `docs/` (次选位置)
3. 项目根目录 (最后选择)

#### 架构文档创建位置顺序
1. `docs/architecture/` (优先标准位置)
2. `docs/` (次选位置)
3. 项目根目录 (最后选择)

#### API文档创建位置顺序
1. `docs/api/` (优先标准位置)
2. `docs/` (次选位置)
3. 项目根目录 (最后选择)

#### 其他文档创建位置顺序
1. `docs/` (优先标准位置)
2. 项目根目录 (备选位置)

### 创建时的文件名规范
- 测试用例：`测试用例文档.md` 或 `测试用例_v{版本号}.md`
- 测试报告：`测试报告_{日期}.md` 或 `测试报告.md`
- 缺陷记录：`缺陷记录.md` 或 `Bug记录.md`
- 开发文档：`开发文档.md` 或 `开发指南.md`
- 说明文档：`说明文档.md` 或 `使用说明.md`

## 五、查找流程图

```
开始查找
  ↓
查找标准位置 (精确匹配)
  ↓
找到? → 是 → 使用该文档
  ↓ 否
查找备选位置 (文件名匹配)
  ↓
找到? → 是 → 使用该文档
  ↓ 否
通配符全局搜索 (多模式尝试)
  ↓
找到? → 是 → 使用该文档
  ↓ 否
在标准位置自动创建文档
  ↓
结束
```

## 七、常见问题

### Q: 找不到文档怎么办？
A: 按照本指南的查找优先级顺序依次尝试，最后在标准位置自动创建。

### Q: 多个匹配文档怎么选择？
A: 选择最新修改的文档，或者文件名最完整匹配的文档。

### Q: 可以在任意位置创建文档吗？
A: 优先在标准位置创建，只有在标准位置不可用时才选择其他位置。
