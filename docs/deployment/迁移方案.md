## 8. Nginx 配置修复方案

**services/nginx/nginx.conf:**

```nginx
http {
    include /etc/nginx/conf.d/*.conf;
    
    upstream backend {
        server auth-service:8001;
        server user-service:8002;
        server project-service:8003;
        server testcase-service:8004;
        server defect-service:8005;
        server requirement-service:8006;
        server dashboard-service:8007;
        server strategy-service:8008;
    }
    
    server {
        listen 80;
        server_name localhost;
        
        location / {
            root /usr/share/nginx/html;
            index index.html;
            try_files $uri $uri/ /index.html;
        }
        
        location /api/ {
            proxy_pass http://backend/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }
    }
}
```

## 13. 风险点分析

| 风险等级 | 风险描述 | 影响范围 | 缓解措施 |
|----------|----------|----------|----------|
| 高 | Docker Compose 路径引用错误 | 所有服务 | 迁移前备份配置文件，迁移后验证路径 |
| 高 | 数据库数据丢失 | 数据层 | 迁移前导出数据库，验证数据完整性 |
| 中 | 服务启动顺序错误 | 服务依赖 | 检查 depends_on 配置 |
| 中 | Nginx 配置错误 | 前端访问 | 备份原配置，测试代理功能 |
| 中 | 环境变量路径错误 | 所有服务 | 验证 .env 文件路径 |
| 低 | 日志目录权限问题 | 日志收集 | 设置正确的目录权限 |
| 低 | 临时文件清理 | 临时数据 | 定期清理 temp 目录 |
