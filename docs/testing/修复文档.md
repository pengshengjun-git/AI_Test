## 一、问题概述

经过对项目前后端代码的全面检查，发现以下模块存在功能问题：
1. **缺陷管理** - 前后端字段不匹配，新增功能无法正常工作
2. **测试计划** - 数据库缺少test_plan表
3. **测试覆盖率** - 数据库缺少test_coverage表
4. **策略管理** - 后端API未处理完整的字段（如projectIds、caseScope、environments等）
5. **需求管理** - 后端API未处理完整的字段
6. **项目管理** - 新增时缺少code字段处理

## 二、详细问题分析

### 2.1 缺陷管理（Defect）

#### 问题描述
前端表单字段与后端API/数据库字段不匹配，导致新增缺陷时数据无法正确保存。

#### 字段对比

| 前端表单字段 | 后端API接收 | 数据库字段 | 状态 |
|-------------|-----------|-----------|------|
| title | title | title | ✅ 匹配 |
| severity | severity | severity | ✅ 匹配 |
| priority | priority | priority | ✅ 匹配 |
| status | status | status | ✅ 匹配 |
| module | module | module | ✅ 匹配 |
| steps (复现步骤) | steps → steps_to_reproduce | steps_to_reproduce | ✅ 已修复 |
| expectedResult | expectedResult → expected_result | expected_result | ✅ 已修复 |
| actualResult | actualResult → actual_result | actual_result | ✅ 已修复 |
| project_id | project_id | project_id | ✅ 匹配 |
| requirement_id | requirement_id | requirement_id | ✅ 匹配 |

#### 修复内容
1. ✅ 前端defect.ts类型定义已更新，添加steps、expectedResult、actualResult字段
2. ✅ 后端API已更新：POST/PUT接口现在正确处理steps→steps_to_reproduce映射
3. ✅ 数据库schema.sql已更新：defect表添加expected_result和actual_result字段

## 六、注意事项

1. **数据库初始化**: 运行修复后的schema.sql会创建新的表和字段，但不会影响已有数据
2. **字段映射规则**:
   - 前端驼峰命名 → 后端蛇形命名（expectedResult → expected_result）
   - 数组字段 → 逗号分隔字符串存储（projectIds: [1,2] → project_ids: "1,2"）
3. **前端兼容性**: 所有前端表单字段保持不变，后端API负责做转换
