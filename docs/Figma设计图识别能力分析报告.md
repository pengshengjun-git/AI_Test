## 二、Figma技能详细介绍

### 2.1 技能工作流程

使用Figma技能的标准流程：

```
1. 获取设计上下文 (get_design_context)
   ↓
2. 获取截图参考 (get_screenshot)
   ↓
3. 下载所需资产
   ↓
4. 开始实施/分析
```

### 2.2 技能核心规则

**必选步骤**：
1. 首先运行 `get_design_context` 获取确切节点的结构化表示
2. 如果响应太大，运行 `get_metadata` 获取高级节点映射
3. 获取正在实施的节点变体的视觉参考
4. 只有同时拥有设计上下文和截图后，才下载所需资产并开始实施

**实现规则**：
- Figma MCP输出（React + Tailwind）作为设计和行为的表示
- 使用项目的颜色系统、字体比例和间距标记
- 与现有路由、状态管理和数据获取模式保持一致
- 力求与Figma设计1:1的视觉一致性

## 四、使用Figma技能的方法

### 4.1 如何提供Figma链接

**正确的Figma链接格式**：
- 复制Figma框架/图层链接
- 链接应指向确切的节点/变体
- 示例：`https://www.figma.com/file/xxxxx/...node-id=xxxxx`

### 4.2 为不同角色定制的使用场景

#### 场景1：产品经理 - 设计图转产品文档
```
输入：Figma设计链接
输出：结构化产品需求文档
内容：
- 功能概述
- 用户流程
- UI规范
- 验收标准
```

#### 场景2：测试工程师 - 设计图转测试用例
```
输入：Figma设计链接
输出：完整测试用例集
内容：
- UI展示测试
- 交互流程测试
- 边界条件测试
- 兼容性测试
```

#### 场景3：开发工程师 - 设计图转代码
```
输入：Figma设计链接
```
