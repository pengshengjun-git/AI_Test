## Brand & Style

This design system is engineered for efficiency, precision, and trust. It targets enterprise QA engineers and DevOps managers who require a high-density information environment that remains legible and calm. The aesthetic is **Corporate Modern**, drawing inspiration from the structured utility of Linear and the collaborative scale of GitLab.

The brand personality is "The Intelligent Co-pilot"—authoritative and reliable, yet forward-thinking. To reflect the AI-powered nature of the platform, the UI uses technical motifs: crisp lines, systematic spacing, and subtle motion. The primary focus is on reducing cognitive load while handling complex data, achieved through a neutral base palette punctuated by a vibrant, "action-oriented" blue.

## Colors

The palette is designed for high-contrast accessibility and long-form readability.

*   **Primary Blue (#007AFF):** Used for primary actions, active navigation states, and key progress indicators. It signifies intelligence and reliability.
*   **Neutral Backgrounds:** A tiered system of grays (`#F8FAFC` for the page background, `#FFFFFF` for cards) creates natural depth without heavy shadows.
*   **Semantic Accents:** Success (Emerald), Warning (Amber), and Error (Rose) are used sparingly for status indicators and automated test results.
*   **AI-Specific Accent:** A secondary Indigo gradient is reserved exclusively for AI-generated features, distinguishing automated insights from manual inputs.

## Typography

This design system utilizes **Inter** for its exceptional legibility in SaaS interfaces and UI density. For technical data—such as test scripts, log outputs, and ID numbers—**JetBrains Mono** provides a clear, tabular distinction.

Headlines use a slight negative letter-spacing to appear tighter and more professional at larger scales. Body text is optimized for the 14px "sweet spot" common in enterprise dashboards, ensuring that large tables and complex forms remain readable.

## Layout & Spacing

The design system employs a **12-column fluid grid** for main content areas, with a max-width of 1440px to prevent excessive line lengths on ultrawide monitors.

*   **Rhythm:** An 8px base grid governs all padding and margins to ensure visual harmony.
*   **Density:** The layout favors "Compact" density for data-heavy views (tables, test suites) and "Default" density for dashboards and settings.
*   **Sidebars:** A fixed 240px left-hand navigation is the standard for top-level app architecture.
*   **Breakpoints:** 
    *   **Desktop:** 12 columns, 24px margins.
    *   **Tablet:** 8 columns, 16px margins, collapsible sidebar.
    *   **Mobile:** 4 columns, 12px margins, bottom sheet navigation for key actions.

## Elevation & Depth

Visual hierarchy is established through **Tonal Layering** supplemented by subtle **Ambient Shadows**. 

*   **Level 0 (Background):** Slate-50 (#F8FAFC). Used for the canvas.
*   **Level 1 (Card/Container):** White (#FFFFFF) with a 1px border (#E2E8F0).
*   **Level 2 (Active/Hover):** Subtle shadow (0px 4px 6px -1px rgba(0, 0, 0, 0.05)) to indicate interactivity.
*   **Level 3 (Modals/Popovers):** Pronounced shadow (0px 10px 15px -3px rgba(0, 0, 0, 0.1)) with a backdrop blur of 8px to focus the user's attention.

Avoid heavy dark shadows; depth should feel like stacked sheets of paper under natural office lighting.

## Shapes

The shape language is strictly **Soft (0.25rem / 4px base)**. While the user requested 8px, this system utilizes a tiered approach where smaller components (buttons, inputs) use 6px and larger containers (cards, modals) use 8px to maintain a balanced "nested" appearance.

*   **Component Radius:** 6px (Small UI elements)
*   **Container Radius:** 8px (Cards, Sections)
*   **Interactive Radius:** 4px (Checkboxes, Tags)

This creates a crisp, professional look that avoids the "playfulness" of overly rounded corners while remaining softer and more modern than sharp edges.

## Components

*   **Buttons:** Primary buttons use a solid #007AFF fill with white text. Secondary buttons use a subtle gray border with slate text. All buttons have a 200ms transition on hover with a slight darkening of the fill.
*   **Inputs:** Default state features a 1px Slate-200 border. Focus state uses a 2px Primary Blue glow (ring).
*   **AI Insights Cards:** Distinguished by a thin 2px gradient border on the left edge and a faint Indigo-50 background tint.
*   **Status Chips:** Small, semi-transparent backgrounds with high-contrast text (e.g., "Passed" uses Emerald-100 background with Emerald-700 text).
*   **Data Tables:** Clean, no vertical borders. Horizontal dividers are 1px Slate-100. Header row uses a subtle Slate-50 background and JetBrains Mono for column titles.
*   **Test Execution Bar:** A sticky bottom component for active test runs, using a blurred glass effect (Glassmorphism) to stay visible over varying content.
