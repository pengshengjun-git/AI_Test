# AI测试管理平台 - 企业级后台管理系统产品需求文档

## 一、业务背景

### 1.1 背景说明
随着AI测试管理平台的逐步落地，企业对系统管理能力、权限控制、数据可视化以及AI调用管理的需求日益增长。当前系统已具备基础的用户认证和RBAC权限能力，但缺少完整的用户管理、角色权限配置、数据看板以及AI调用监控等企业级功能。

### 1.2 市场机遇
- 企业级SaaS产品对系统管理、权限控制的要求日益严格
- 数据可视化和统计分析成为产品竞争力的重要组成部分
- AI服务使用量监控和成本管理是企业关注的重点
- 完整的系统管理功能是产品从MVP走向企业级的必经之路

### 1.3 解决的痛点
- 缺少统一的用户管理界面，无法方便地创建、编辑、禁用用户
- 缺少可视化的菜单权限配置功能
- 缺少数据看板，无法直观地查看项目和测试数据
- 缺少AI调用记录和统计，无法监控AI服务使用情况和成本

## 四、数据与约束

### 4.1 数据来源与格式
- 用户数据：存储在 MySQL `user` 表
- 角色数据：存储在 MySQL `role` 表
- 菜单数据：存储在 MySQL `menu` 表
- AI调用数据：存储在 MySQL `ai_call_record` 表
- 项目/测试数据：来自各业务服务

### 4.2 数据更新频率
- 用户/角色/菜单：实时更新
- AI调用记录：实时写入，定期统计
- 看板数据：5分钟刷新一次

### 4.3 技术约束
| 约束项 | 要求 |
|-------|------|
| 响应时间 | API 响应 < 200ms |
| 并发支持 | 支持 100+ 并发用户 |
| 可用性 | 系统可用性 99.9% |
| 安全性 | 完整的JWT认证和RBAC权限控制 |

### 4.4 成本约束
- 开发周期：3-4周完成P0功能
- 资源投入：前后端各1-2人
- AI成本：通过记录和统计优化AI调用成本

### 4.5 合规要求
- 用户密码加密存储
- 敏感数据脱敏
- 操作日志可追溯
- 符合数据安全要求

## 五、AI能力边界

### 5.1 AI能力范围
本次需求不涉及新增AI能力，主要是**AI调用数据的记录和统计**：
- ✅ 记录每次AI调用详情
- ✅ 统计AI调用次数、Token使用量、成本
- ✅ 提供AI调用趋势分析
- ❌ 不涉及新增AI功能
- ❌ 不涉及AI能力优化

### 5.2 异常处理与人工介入
- AI调用失败时记录错误信息，不影响系统运行
- AI调用成本超阈值时发出告警提示
- 详细的调用记录便于问题排查

## 七、流程设计

### 7.1 用户管理流程

#### 7.1.1 创建用户流程
```
管理员进入用户管理页面
    ↓
点击"新建用户"按钮
    ↓
填写用户基本信息（用户名、邮箱、密码等）
    ↓
选择用户角色
    ↓
点击"保存"
    ↓
系统验证信息有效性
    ↓
验证通过 → 用户创建成功，发送激活邮件
验证失败 → 提示错误信息，返回编辑
```

#### 7.1.2 禁用/启用用户流程
```
管理员在用户列表找到目标用户
    ↓
点击"禁用"或"启用"按钮
    ↓
二次确认
    ↓
系统更新用户状态
    ↓
禁用用户 → 用户无法登录，收到通知
启用用户 → 用户可以正常登录
```

### 7.2 角色权限配置流程

```
管理员进入角色管理页面
    ↓
选择或创建目标角色
    ↓
点击"菜单权限"配置
    ↓
打开菜单树，勾选需要的菜单权限
    ↓
点击"保存"
    ↓
系统保存角色菜单关联
    ↓
拥有该角色的用户登录后看到对应菜单
```

### 7.3 数据看板访问流程

```
用户登录系统
    ↓
进入数据看板页面
    ↓
系统根据用户权限获取数据范围
    ↓
加载并展示概览数据
    ↓
用户可切换标签查看项目/测试统计
    ↓
用户可调整时间范围筛选数据
```

### 7.4 AI调用记录流程

```
用户使用AI功能（如生成用例）
    ↓
系统记录调用信息到 ai_call_record 表
    ↓
包括：用户ID、功能类型、模型、Token数、状态、成本等
    ↓
管理员可在AI工具箱查看调用记录
    ↓
进行统计分析和成本管控
```

## 十、数据流

### 10.1 用户/角色/菜单数据流
```
前端请求（含JWT）
    ↓
JWT Filter 验证并提取用户信息
    ↓
Controller 接收请求
    ↓
Service 处理业务逻辑
    ↓
Mapper 读写数据库
    ↓
返回结果给前端
```

### 10.2 数据看板数据流
```
用户请求看板数据
    ↓
Dashboard Service 接收请求
    ↓
Feign 调用各业务服务获取数据
    ↓
聚合统计数据
    ↓
返回给前端展示
```

### 10.3 数据存储策略
- 用户/角色/菜单：持久化存储到 MySQL
- AI调用记录：持久化存储到 MySQL，支持归档
- 看板统计：实时计算，不持久化

## 十二、异常兜底

### 12.1 数据库异常
- **场景**：数据库连接失败、查询超时
- **处理**：返回友好错误提示，记录详细错误日志
- **重试**：自动重试1次，失败后人工介入

### 12.2 权限校验异常
- **场景**：用户访问无权限的功能
- **处理**：返回403 Forbidden，提示权限不足
- **记录**：记录到操作日志

### 12.3 数据看板加载失败
- **场景**：统计数据加载失败
- **处理**：展示友好提示，提供"重试"按钮
- **降级**：展示缓存的旧数据
