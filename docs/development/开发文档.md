# 开发指南

## 项目概述

AI测试管理平台是一个基于微服务架构的智能测试管理系统，集成了AI能力实现测试用例自动生成、需求智能解析、缺陷分析等功能。

## 技术栈

### 前端
- Vue 3 + TypeScript
- Vite
- Pinia (状态管理)
- Element Plus (UI组件库)
- Vue Router (路由管理)

### 后端
- Spring Boot 3.2
- Java 21
- MyBatis Plus
- MySQL 8.0
- Redis 7

### AI服务
- FastAPI
- Python 3.11
- LangChain

## 服务说明

### 1. auth-service (8001)
**功能**: 用户认证、登录、注册、Token管理

**主要接口**:
- `POST /api/v1/auth/login` - 用户登录
- `POST /api/v1/auth/register` - 用户注册
- `POST /api/v1/auth/logout` - 用户登出

**特点**: 
- 不使用Redis和Nacos
- 使用JWT Token进行身份验证

### 2. user-service (8002)
**功能**: 用户信息管理、角色权限管理

**主要接口**:
- `GET /api/v1/user/info` - 获取用户信息
- `GET /api/v1/users` - 用户列表
- `PUT /api/v1/users/{id}` - 更新用户信息

### 3. project-service (8003)
**功能**: 项目管理、项目成员管理

**主要接口**:
- `GET /api/v1/projects` - 项目列表
- `POST /api/v1/projects` - 创建项目
- `PUT /api/v1/projects/{id}` - 更新项目
- `DELETE /api/v1/projects/{id}` - 删除项目

### 4. testcase-service (8004)
**功能**: 测试用例管理、用例执行

**主要接口**:
- `GET /api/v1/testcases` - 用例列表
- `POST /api/v1/testcases` - 创建用例
- `POST /api/v1/testcases/{id}/execute` - 执行用例
- `POST /api/v1/testcases/generate` - AI生成用例

### 5. defect-service (8005)
**功能**: 缺陷管理、缺陷跟踪

**主要接口**:
- `GET /api/v1/defects` - 缺陷列表
- `POST /api/v1/defects` - 创建缺陷
- `PUT /api/v1/defects/{id}` - 更新缺陷
- `POST /api/v1/defects/{id}/assign` - 分配缺陷

### 6. requirement-service (8006)
**功能**: 需求管理、需求解析

**主要接口**:
- `GET /api/v1/requirements` - 需求列表
- `POST /api/v1/requirements` - 创建需求
- `POST /api/v1/requirements/{id}/parse` - AI解析需求
- `POST /api/v1/requirements/upload` - 上传需求文档

**特点**: 
- 支持文件上传（最大50MB）
- 本地存储或MinIO存储

### 7. dashboard-service (8007)
**功能**: 工作台统计数据

**主要接口**:
- `GET /api/v1/dashboard/stats` - 获取统计数据
- `GET /api/v1/dashboard/recent-projects` - 最近项目
- `GET /api/v1/dashboard/recent-defects` - 最近缺陷
- `GET /api/v1/dashboard/system-status` - 系统状态
- `GET /api/v1/dashboard/todo-list` - 待办事项
- `PUT /api/v1/dashboard/todo/{id}/status` - 切换待办状态

**特点**: 
- 不使用Redis和Nacos
- 提供实时统计数据

### 8. strategy-service (8008)
**功能**: 测试策略管理

**主要接口**:
- `GET /api/v1/strategy` - 策略列表
- `POST /api/v1/strategy` - 创建策略
- `PUT /api/v1/strategy/{id}` - 更新策略
- `DELETE /api/v1/strategy/{id}` - 删除策略
- `PUT /api/v1/strategy/{id}/status` - 切换策略状态

**策略类型**:
- SMOKE: 冒烟测试
- REGRESSION: 回归测试
- PERFORMANCE: 性能测试
- SECURITY: 安全测试
- INTEGRATION: 集成测试

### 9. ai-service (8010)
**功能**: AI能力服务

**主要接口**:
- `POST /api/v1/ai/generate/testcases` - AI生成测试用例
- `POST /api/v1/ai/parse/requirement` - AI解析需求
- `POST /api/v1/ai/analyze/defect` - AI分析缺陷
- `POST /api/v1/ai/generate/report` - AI生成报告

## 本地开发

### 前置要求
- Node.js 18+
- Java 21
- Maven 3.9+
- Python 3.11
- MySQL 8.0
- Redis 7

### 启动基础设施

```bash
cd docker
docker compose up -d mysql redis
```

### 启动后端服务

```bash
cd backend

# 编译所有模块
mvn clean install -DskipTests

# 启动各个服务（在不同终端窗口）
cd auth-service && mvn spring-boot:run
cd user-service && mvn spring-boot:run
cd project-service && mvn spring-boot:run
cd testcase-service && mvn spring-boot:run
cd defect-service && mvn spring-boot:run
cd requirement-service && mvn spring-boot:run
cd dashboard-service && mvn spring-boot:run
cd strategy-service && mvn spring-boot:run
```

### 启动AI服务

```bash
cd ai-service
pip install -r requirements.txt
python main.py
```

### 启动前端

```bash
cd frontend
npm install
npm run dev
```

访问: http://localhost:3000

## 常见问题

### 1. 端口冲突
修改 `application.yml` 或 `docker-compose.yml` 中的端口配置

### 2. 数据库连接失败
检查MySQL是否启动，用户名密码是否正确

### 3. Redis连接失败
检查Redis是否启动，配置文件中的host和port是否正确

### 4. 前端无法访问后端
检查vite.config.ts中的proxy配置是否正确
